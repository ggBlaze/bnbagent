# BNB Agent — x402 Pay-Per-Request

x402 is the open protocol that uses HTTP 402 to let an agent pay for API calls
in USDC stablecoins, with no API key, no KYC, no subscription. BNB Agent uses
x402 to pay for **every** CoinMarketCap Data API call (in v2.0 also for
TokenModule metadata enrichment).

## The flow

```
Agent (client)                 CoinMarketCap (server)              BSC RPC
   │                                  │                              │
   │   GET /v1/quotes/latest?…        │                              │
   │ ───────────────────────────────> │                              │
   │                                  │                              │
   │   402 Payment Required           │                              │
   │   X-PAYMENT-REQUIRED: <b64>      │                              │
   │ <─────────────────────────────── │                              │
   │                                  │                              │
   │   Parse requirements:            │                              │
   │     network: bsc                 │                              │
   │     token: 0x8AC76a51... (USDC)  │                              │
   │     amount: 10000 (=$0.01)       │                              │
   │     payTo: 0x...                 │                              │
   │     nonce: ...                   │                              │
   │   EIP-3009 sign transferWith-   │                              │
   │   Authorization(USDC)            │                              │
   │ ──────────────────────────────────────────────────────────────> │
   │                                  │   USDC.transferWithAuth()    │
   │                                  │   (signed; <200ms finality)  │
   │   GET /v1/quotes/latest?…        │                              │
   │   X-PAYMENT: <b64>               │                              │
   │ ───────────────────────────────> │                              │
   │                                  │                              │
   │   200 OK + data                  │                              │
   │ <─────────────────────────────── │                              │
```

## Why EIP-3009?

EIP-3009 `transferWithAuthorization` is a gasless, signed USDC transfer. The
agent signs once, the server submits the on-chain transfer, and the funds move
with <200ms finality on BNB Chain. No gas for the agent — the server pays.

## The implementation

See [`connectors/x402.py`](../connectors/x402.py). The two key functions:

```python
def decode_payment_requirements(b64_header: str) -> PaymentRequirements
async def x402_pay(required_b64: str, wallet, chain_id: int = 56) -> str
```

The agent wraps every CMC call with `x402_pay`:

```python
async def cmc_call(method, path, params):
    r = await httpx.request(method, f"{BASE}{path}", params=params)
    if r.status_code != 402:
        return r.json()
    payment_hdr = await x402_pay(r.headers["X-PAYMENT-REQUIRED"], wallet)
    r2 = await httpx.request(method, f"{BASE}{path}", params=params,
                              headers={"X-PAYMENT": payment_hdr})
    return r2.json()
```

## The daily spend cap

The policy has a `fees.x402_max_usdc_per_day` field (default $10). The agent
tracks today's spend in `CMCClient.spend_today` and refuses any CMC call that
would exceed the cap. Judges see this in the dashboard's `cmc-spend-pill`.

## v2.0 uses

The same x402 mechanism is used in two more places:

- **TokenModule** — when deploying a token, the agent calls
  `cmc.call("GET", "/v1/cryptocurrency/info", ...)` to enrich the token
  metadata (name + symbol cross-check, existing-CMC entry, etc). The
  resulting microcharge shows up in the same `/api/cmc-charges` ledger.
- **MCP server** — `bnbagent_deploy_token` triggers the same chain
  end-to-end, so token deploys initiated by other agents also pay
  for CMC metadata.

## Demo evidence

## Why this wins the **Best Use of CoinMarketCap** prize

1. **Deep API usage** — Data API + Data MCP + Skills marketplace + x402
2. **Production x402 integration** — the agent pays for its own data, not via API key
3. **Visible evidence** — every microcharge is on the dashboard with BscScan links
4. **Spend-controlled** — daily cap enforced by the policy
5. **Composable with Skills + TokenModule** — the same x402 flow drives
