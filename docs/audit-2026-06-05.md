# BNB Agent — Trading Logic Audit · 2026-06-05

This is the audit + hardening pass that was performed before the live
PnL-replay window. It is preserved here for transparency — judges and
operators can see exactly what was checked and what was changed.

## Scope

- `core/portfolio.py` — equity, peak, drawdown, PnL
- `core/risk.py` — circuit-breaker, Kelly, cap-by-risk
- `core/tick.py` — sleeve harness, heartbeat, allow_trade
- `strategies/sleeve_a_carry.py` — funding carry
- `strategies/sleeve_b_momentum.py` — DEX momentum
- `strategies/sleeve_c_meanrev.py` — mean-reversion
- `backtest/replay.py` — replay harness

## Findings + fixes

| # | File | Finding | Fix |
|---|---|---|---|
| 1 | `core/portfolio.py` | `mark_to_market` for `spot_long+perp_short` returned only funding income, ignoring spot PnL vs current mark. Carry would look flat even when the spot leg moved. | Spot PnL added to funding income. Perp short leg is delta-neutral by construction. |
| 2 | `core/portfolio.py` | `day_start_equity` was only seeded after the first `update_peak()` call, so the daily-loss breaker was inactive on day 0. | Seeded in `__init__` from `_today_static()`. |
| 3 | `core/risk.py` | No kill-switch path — once the agent was running, the only way to stop new orders was to kill the process. | New check `policy._kill_switch` at the top of `circuit_breaker_check`. Set by the dashboard. |
| 4 | `strategies/sleeve_a_carry.py` | `_collect_funding` accrued funding **every 30-second tick** instead of on 8h boundaries. This over-credited carry PnL by ~960×. | Now gated on `now - last_funding_accrual >= 8h`. |
| 5 | `strategies/sleeve_a_carry.py` | Basis-trigger exit called `perps.close_short` directly but did not close the spot leg or remove the position from the portfolio — leaving a phantom spot position. | Routed through `_close_pair` so both legs close and the portfolio is updated atomically. |
| 6 | `strategies/sleeve_b_momentum.py` | Re-entry into a symbol was allowed immediately after a stop-out, which can become a "revenge trade" loop in choppy markets. | Added `loss_cooldown_until` map; 4h cool-off after a losing exit. |
| 7 | `strategies/sleeve_c_meanrev.py` | Same revenge-trade risk as Sleeve B. | Same fix; 6h cool-off (longer because z-score signals are sparser). |
| 8 | `strategies/{a,b,c}.py` | `_token_address` was duplicated across the 3 sleeves. | Extracted to `core/utils.py::token_address`. |
| 9 | `backtest/replay.py` | The mark-price lambda captured `ts` by name from the loop, so the closure was reading the **final** ts in the loop, not the per-iteration ts. | Rebound with default-arg `lambda s, _ts=ts: ...` so each iteration gets a fresh binding. |
| 10 | `core/risk.py` | Risk check accepted any `proposed.is_new` value — could be `None` and crash the per-trade risk calculation. | Guarded. |

## Verification

- All 64+ unit + integration tests pass.
- `python -m backtest.replay` runs end-to-end; report at `data/reports/replay.html`.
- Kill switch: `POST /api/control {"kill": true}` → next `allow_trade` returns
  `False, "kill switch engaged: …"`.
- Funding accrual: Sleeve A `_collect_funding` now exits immediately unless
  8h has elapsed — visible in the log when it actually fires.

## What we did NOT change

- **Sleeve strategy logic itself** (entry signals, ATR / z-score thresholds,
  Kelly fraction). These are the alpha and were tuned in the backtest. The
  fixes above are safety / correctness — they tighten the harness, not the
  strategy.
- **Risk caps** in `policy.yaml` (1% per-trade, 3% daily, 2x leverage,
  15% single-position). These are signed and were not touched.
- **Allowlist**. Hard-coded to the curated top-20 BSC tokens.

## v2.0 audit pass (AI Agent Team)

The original audit covered the trading engine. v2.0 added the AI agent
team, the Token Module, the Skills registry, and the MCP server. The
same rigor was applied to the new code.

| # | File | Finding | Fix |
|---|---|---|---|
| 11 | `agents/advisor.py` | If a hostile or buggy LLM response tries to **raise** a cap, the agent would over-write the policy. | `_apply()` enforces `new < old` for every numeric cap. Test `test_cannot_loosen_with_higher_value` locks the invariant. |
| 12 | `agents/reviewer.py` | A slow LLM call could block the sleeve tick (Sleeve A ticks every 30s, Sleeve B/C every 5min — but CPU is still tight). | `asyncio.wait_for(..., timeout=0.5)`. Fallback to `_heuristic_veto` always works in <1ms. Test `test_llm_timeout_falls_back_to_heuristic` locks this. |
| 13 | `agents/reviewer.py` | LLM saying `allow=true` with a high confidence could still pass a dangerous trade if the sleeve was already in drawdown. | Post-LLM `_heuristic_veto` overlay checks: sleeve_dd > 50% of cap, win_rate < 20%, post-loss cooldown, 4/5 recent losses. |
| 14 | `agents/chat.py` | The chat could accidentally recommend loosening the policy. | `recommend_risk_change` returns a recommendation + a UI prompt to Setup; it never writes. Test `test_recommend_risk_change_does_not_write` locks this. |
| 15 | `agents/chat.py` | The chat could enable a Skill with missing API keys. | `enable_skill` goes through `SkillRegistry`, which checks `requires` env vars before enable. Test `test_enable_missing_env_blocks` locks this. |
| 16 | `agents/token_module.py` | Generated landing-page HTML could contain `eval(...)` / `document.write(...)` / external scripts. | Server-side sanitizer strips these patterns before returning HTML. Tests `test_sanitize_website_strips_eval` + `_strips_external_script_src` + `_strips_event_handlers`. |
| 17 | `agents/token_module.py` | A typo in the token name on a **mainnet** deploy would burn real BNB + lock in the wrong contract name forever. | `confirm_mainnet: true` in the API + dashboard `<dialog>` requires the user to type the token name. Test `test_mainnet_requires_explicit_network`. |
| 18 | `connectors/bnb_sdk.py` | Testnet `BSCClient.broadcast` returned `contract_address=None`, so the TokenModule demo could not show a deploy address. | Stub now derives a deterministic address from `keccak(sender || nonce)[:20]`. Test `test_create_token_returns_valid_address`. |
| 19 | `core/portfolio.py` | `sleeve_exposure(sleeve)` required a sleeve arg; the advisor needed an all-sleeves view. | Added `sleeve_exposures()` returning `{sleeve: total_notional}`. |
| 20 | `agent_mcp/mcp_server.py` | The MCP server's mainnet guard was checked **after** the TokenModule check, so a missing TokenModule hid the real "confirm_mainnet" error. | Reordered: confirm_mainnet is checked first. Test `test_deploy_token_mainnet_without_confirm_rejected`. |
